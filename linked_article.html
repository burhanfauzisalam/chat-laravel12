Howdy, IT fellows!

AI isn’t just for tech giants or big cloud servers anymore — it’s coming straight to your laptop or local setup. Yup, you can now run powerful AI models right on your own machine and use them to do some seriously cool stuff, like talking to a database in plain English.

Forget writing complicated SQL queries or digging through endless spreadsheets. You just ask your question, and boom — the AI fetches the info and breaks it down for you like a pro.

Imagine saying something like,

Hey, what were our best-selling items last month?

or

Hello, can you tell me which customers made the most orders in the last 3 months?

Instead of needing a data team to figure that out, AI handles it in seconds — giving you clear answers or even charts that make it super easy to get what’s going on.

Even better, this isn’t just about looking stuff up. Local AI can actually analyze the data, spot trends, flag weird stuff, and even give you tips on what to do next. And since it’s all running locally, your data stays safe and private. Whether you're a startup, a solo dev, or just someone who wants smarter tools without selling your soul to the cloud, local AI is a total game changer.

In this article, you’ll figure out how to :

  • Install OS Updates & System Dependencies needed
  • Use some Flask functions
  • Use PyMySQL ( I am using MySQL database, if you are using other database engine, make sure you have correct library )
  • Use Python binding Llama.cpp
  • Create service to automatically run your application
  • Create simple UI for chatting with the AI

Okay, first let's get on the same page. I made this article especially for companies with not too big resources such as SMEs or small to medium-sized companies.

In this article, I use Python binding llama.cpp and qwen2-7b-instruct-q4_0 model in GGUF format provided by Qwen on the HuggingFace website. This model has Lower Quantization for Better CPU Performance (so you don't need any high-cost GPU to run this model), but you can choose another model that fit your hardware configurations.

To find out more about what GGUF is, you can see it on this link :

Medium - Understanding the GGUF Format (A Comprehensive Guide)

  2. I want all servers to be in the local environment
  3. Because this project is for small and medium businesses, I am looking for a model that does not require large resources
  4. Can run on CPU without needing a GPU

You can choose which model suits with your hardware configurations. Here is the syntax to download the model : 

mkdir models
cd models
wget https://huggingface.co/Qwen/Qwen2-7B-Instruct-GGUF/resolve/main/qwen2-7b-instruct-q4_0.gguf

Please note that the syntax above will require approximately 4.44 GB of space on your hard disk.

Also note that I downloaded the model with 4-bit quantization to reduce the size of large models, significantly lowering their memory footprint while maintaining adequate precision for inference.

Feel free to choose the type of quantization that you feel suits your hardware configuration.

Function to get database schema for prompt context

def get_schema():
    try:
        with get_db_connection() as conn:
            with conn.cursor() as cursor:
                cursor.execute("SHOW TABLES")
                tables = [row['Tables_in_' + db_config['database']] for row in cursor.fetchall()]
                schema = {}
                for table in tables:
                    cursor.execute(f"DESCRIBE {table}")
                    columns = cursor.fetchall()
                    schema[table] = [{'Field': col['Field'], 'Type': col['Type']} for col in columns]
                return schema
    except pymysql.Error as e:
        logger.error(f"Failed to retrieve schema: {str(e)}")
        raise

The snippet above runs a DDL command "SHOW TABLES" to get information about what tables are in the database and some information about those tables.

And then, for each table, it runs a DDL command "DESCRIBE [table name]" to get Table Structure Information.

And finally save the table information in a variable which contains each field's name and data type.

Function to generate SQL query from natural language

def generate_sql_query(nl_query, schema):
    schema_str = json.dumps(schema, indent=2, ensure_ascii=False)
    prompt = f"""
Anda adalah asisten AI yang mengubah pertanyaan dalam bahasa Indonesia menjadi kueri SQL. Berikut adalah skema database:
{schema_str}

Pertanyaan: {nl_query}

Buat kueri SQL yang sesuai untuk menjawab pertanyaan tersebut. Kembalikan hanya kueri SQL dalam format teks biasa, tanpa penjelasan tambahan.
"""
    try:
        response = llm(
            prompt,
            max_tokens=256,
            temperature=0.1,  # Low temperature for deterministic output
            stop=["\n\n"]  # Stop at double newline to avoid extra text
        )
        sql_query = response['choices'][0]['text'].strip()
        logger.info(f"Generated SQL query: {sql_query}")
        return sql_query
    except Exception as e:
        logger.error(f"Failed to generate SQL query: {str(e)}")
        return None

The above function will task LLM to create an SQL query from the user's question along with the previously generated database schema. Note that there is an important point to understand in the snippet above.

schema_str = json.dumps(schema, indent=2, ensure_ascii=False)

The parameter "ensure_ascii=False" ensures that we will get the database schema with the characters as is if the characters are Non-ASCII characters, or we are dealing with UTF-8 encoded JSON strings or internationalized data that should be displayed correctly across different systems.

Output example : {"name": "你好", "city": "São Paulo", "country": "대한민국"}

What am I doing here? I am "adding" information to LLM so that it knows the "relevant" information and prevent the LLM not doing any "hallucination". By informing the schema of my database, I have done what is called RAG (Retrieval-Augmented Generation).

Here are some interesting articles about RAG that you can read to deepen your understanding of what RAG is:

  • NVidia Blog - What Is Retrieval-Augmented Generation, aka RAG?
  • Wikipedia - Retrieval-augmented generation
  • Google Cloud - What is Retrieval-Augmented Generation (RAG)?

Function to execute SQL query

def execute_sql_query(sql_query):
    try:
        with get_db_connection() as conn:
            with conn.cursor() as cursor:
                cursor.execute(sql_query)
                results = cursor.fetchall()
                conn.commit()
                logger.info(f"SQL query executed successfully, {len(results)} rows returned")
                return results
    except pymysql.Error as e:
        logger.error(f"Failed to execute SQL query: {str(e)}")
        return None

The above snippet will execute whatever SQL query is received on the "sql_query" parameter variable.

API endpoint to handle natural language queries

@app.route('/api/nl2sql', methods=['POST'])
def nl2sql():
    try:
        data = request.get_json()
        if not data or 'query' not in data:
            return jsonify({'error': 'Missing query in request'}), 400

        nl_query = data['query']
        logger.info(f"Received natural language query: {nl_query}")

        # Get database schema
        schema = get_schema()

        # Generate SQL query
        sql_query = generate_sql_query(nl_query, schema)
        # logger.info(f"[DEBUG] sql_query: {sql_query}") # uncomment for DEBUG if error executing sql query
        sqlClean = re.sub('<think>?(.*?)</think>', '', sql_query, flags=re.DOTALL).strip()
        sqlClean = re.sub('Untuk menjawab pertanyaan tersebut, kita perlu?(.*?)Kueri SQL yang sesuai adalah: ', '', sqlClean, flags=re.DOTALL).strip()
        sqlClean = re.sub('Kueri SQL: ', '', sqlClean, flags=re.DOTALL).strip()
        sqlClean = re.sub('sql\n', '', sqlClean, flags=re.DOTALL).strip()
        sqlClean = re.sub('`', '', sqlClean, flags=re.DOTALL).strip()
        sql_query = sqlClean
        if not sql_query:
            return jsonify({'error': 'Failed to generate SQL query'}), 500

        # Execute SQL query
        results = execute_sql_query(sql_query)
        if results is None:
            return jsonify({'error': 'Failed to execute SQL query'}), 500

        return jsonify({
            'sql_query': sql_query,
            'results': results
        }), 200

    except Exception as e:
        logger.error(f"Error processing request: {str(e)}")
        return jsonify({'error': str(e)}), 500

The above snippet will be executed when the API is called with the route "/api/nl2sql" using "POST" method. The thing to note here is that there are several checks on the query results using the "sqlClean" variable which cleans the received query results if there are still inappropriate characters, or if the LLM produces unnecessary explanations or any additional words.

If you encounter an error when running this program with the error "Failed to execute SQL query", then DEBUG the program to see what the output of the "sql_query" variable is after the line: sql_query = generate_sql_query(nl_query, schema)

Database design

For the purpose of this article, I have created a new database with some dummy data in it. Here is its ERD.

As you can see in the ERD above, this Sample Database has several relationships between tables. However, with the implementation of RAG, LLM is "expected" to be able to understand the structure of this database and use that understanding to derive a suitable SQL query.

In this database, I use English fields naming to make it easier for LLM to "understand" the structure without me having to explain it in the "description" section of each field. If you use non-standard field naming, then you need to add an explanation in the description section of each field so that LLM can understand the contents of the field.

You can also "train" LLM by adding some sample queries to the prompt section if necessary, for example like this: 

Contoh pertanyaan : Berapa jumlah pegawai yang dibuat dalam minggu terakhir?
Hasil kueri SQL : SELECT COUNT(*) FROM employees WHERE created_date >= DATE_ADD(CURDATE(), INTERVAL 7 DAY)

Add the above explanation to the prompt variable in the generate_sql_query function.

        // Function to set a cookie
        function setCookie(name, value, days) {
            let expires = "";
            if (days) {
                const date = new Date();
                date.setTime(date.getTime() + (days * 24 * 60 * 60 * 1000));
                expires = "; expires=" + date.toUTCString();
            }
            document.cookie = name + "=" + (value || "") + expires + "; path=/";
        }

        // Function to get a cookie
        function getCookie(name) {
            const nameEQ = name + "=";
            const ca = document.cookie.split(';');
            for (let i = 0; i < ca.length; i++) {
                let c = ca[i];
                while (c.charAt(0) === ' ') c = c.substring(1, c.length);
                if (c.indexOf(nameEQ) === 0) return c.substring(nameEQ.length, c.length);
            }
            return null;
        }

        // Function to load question history from cookies
        function loadHistory() {
            const history = getCookie('queryHistory');
            if (history) {
                return JSON.parse(history) || [];

            return [];

        function saveToHistory(query) {

        document.getElementById('queryForm').addEventListener('submit', async (e) => {
            e.preventDefault();
            const query = document.getElementById('query').value.trim();
            if (!query) return;

            saveToHistory(query);

            const responseDiv = document.getElementById('response');
            responseDiv.classList.remove('hidden');
            document.getElementById('sqlQuery').textContent = 'Memproses...';
            document.getElementById('results').textContent = '';

            try {
                const response = await fetch('https://aisvr:5000/api/nl2sql', {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify({ query })
                });
                const data = await response.json();

                if (response.ok) {
                    document.getElementById('sqlQuery').textContent = data.sql_query || 'Tidak ada kueri SQL dihasilkan';
                    document.getElementById('results').textContent = JSON.stringify(data.results, null, 2);
                } else {
                    document.getElementById('sqlQuery').textContent = 'Error';
                    document.getElementById('results').textContent = data.error || 'Terjadi kesalahan';
                }
            } catch (error) {
                document.getElementById('sqlQuery').textContent = 'Error';
                document.getElementById('results').textContent = 'Gagal terhubung ke server: ' + error.message;
            }
        });

        window.onload = displayHistory;
    </script>
</body>
</html>

Look for the text: "https://aisvr:5000/api/nl2sql" and replace it with your API backend server address. What needs to be noted here is, if your backend API runs securely using https, then make sure that the origin of the web server serving this UI has the appropriate CA certificate. Creating Automatic Service for the Backend API

Now that we have successfully created this application, we need to make sure that this Backend API will always run on the server. The way to do this is by creating a service on the server that will be loaded every time the server is turned on. Here is the syntax: 

cd ~
nano start_backend.sh

We will create a file that contains all the commands that must be executed to run our application. The contents of the "start_backend.sh" file is: 

#!/bin/bash
cd /home/[yourusername]/pyapp
source ~/py_envs/bin/activate
python api.py

So basically, we run this application by first creating a virtual environment for python. Note: please replace the text "[yourusername]" with your own username. After you save the file, continue to try to run the file by running the command below: 

chmod +x start_backend.sh 
./start_backend.sh

Creating the service

Now it's time to create a service on your OS by running the commands below: 

sudo nano /etc/systemd/system/api_backend.service

This command will create a new service named "api_backend.service". You can copy the contents of the file from the snippet below: 

[Unit]
Description=Flask + Llama Backend
After=network.target

[Service]
User=[yourusername]
WorkingDirectory=/home/[yourusername]/pyapp
ExecStart=/home/[yourusername]/start_backend.sh
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target

Note: please replace the text "[yourusername]" with your own username. To run the service that we have created above, run these commands: 

sudo systemctl daemon-reexec
sudo systemctl daemon-reload
sudo systemctl enable api_backend.service
sudo systemctl start api_backend.service
sudo systemctl status api_backend.service

To view the service status or to perform debugging, run the command below: 

journalctl -u api_backend.service -f

And there you have it.
